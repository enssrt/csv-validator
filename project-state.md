## Принятые архитектурные решения
- Внешние пути запуска задаются только через CLI.
- Внутренние бизнес-правила валидации живут в config.py.
- Валидация значений вынесена в отдельный модуль validators.py.
- Ошибки row-level валидации не получают отдельный статус и входят в ERROR.
- Проект развивается через один активный проект, а не через переключение на новый.
