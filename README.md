# CSV Validator Pipeline

Утилитарный инструмент на Python для автоматического анализа, валидации и обработки CSV-таблиц с генерацией отчетов.

## Функционал
* Сканирование входной директории и пакетная обработка файлов.
* Валидация структуры CSV (проверка обязательных колонок из конфигурации).
* Логирование результатов и сбор итоговой статистики.
* Автоматическая генерация отчетов по результатам обработки.

## Статусы обработки файлов
* `OK` — Файл успешно прошел валидацию и обработан.
* `ERROR` — Файл содержит ошибки структуры (например, не хватает колонок).
* `NOT_READ` — Не удалось прочитать файл (ошибка доступа или поврежденный файл).
* `REPORT_ERROR` — Файл валиден, но произошла системная ошибка при попытке записать отчет.

## Структура проекта
* `main.py` — Точка входа, инициализация параметров запуска и запуск пайплайна (`run_pipeline`).
* `source/` — Исходный код логики приложения:
  * `processor.py` — Основной конвейер обработки одиночных файлов и папок.
  * `validators.py` — Модули (юниты) построчной валидации бизнес-параметров.
  * `csv_utils.py` / `report_utils.py` — Утилиты для чтения таблиц и генерации текстовых отчетов.
  * `config.py` — Конфигурационные константы проекта.
* `tests/` — Раздельное тестовое покрытие: интеграционные сценарии процессора (`test_file_processor.py`) и изолированные юнит-тесты валидаторов (`test_validators.py`).
* `requirements.txt` — Список внешних зависимостей проекта.

## Архитектура и конфигурация (v2)

В текущей версии проекта реализовано строгое разделение между внешним интерфейсом управления, конвейером обработки и внутренними правилами бизнес-логики:

* **Интерфейс запуска (CLI слой):** Входная директория с данными (`--input`) и директория для сохранения отчетов (`--output`) являются динамическими параметрами. Они передаются пользователем через аргументы командной строки при каждом запуске утилиты. Если обязательные флаги не указаны, программа завершит работу с кодом ошибки `2`.
* **Бизнес-правила (Business rules слой):** Внутренние правила валидации изолированы от интерфейса. Список обязательных колонок зафиксирован в файле `source/config.py` в виде константы `REQUIRED_COLUMNS`. Это исключает случайные ошибки ввода в терминале и защищает логику анализатора.
* **Коды завершения (Exit Codes):** Программа возвращает код `0` при успешной обработке существующей директории и код `1` при критических ошибках инициализации (например, если исходная папка не найдена).

## Двухуровневая валидация данных (Версия v2)

Конвейер выполняет проверку файлов строго последовательно. Построчный анализ запускается только тогда, когда файл успешно прошел проверку структуры.

### 1. Структурная проверка (Structural Check)
Проверяется наличие нормализованных обязательных колонок, заданных в конфигурации. Если структура нарушена, файл сразу получает статус `ERROR`, а построчный цикл в целях экономии ресурсов пропускается. В отчет записывается предупреждение: *«Валидация строк не производилась из-за критической ошибки структуры»*.

### 2. Построчная валидация (Row-level Validation)
Если структура корректна, конвейер инициализирует построчный цикл (начиная со 2-й строки файла) и валидирует следующие поля:
* **Цена (`price`)**:
    * Значение обязано быть числовым (целое или с плавающей точкой).
    * Бизнес-логика: цена должна быть строго больше нуля. Любые отрицательные значения или ноль переводят строку в разряд невалидных.
* **Дата (`date`)**: Строка должна строго соответствовать международному стандарту `YYYY-MM-DD`, а сама дата должна быть календарно существующей (система отлавливает несуществующие дни вроде 31 июня или 13-й месяц).

При обнаружении ошибок строк они добавляются в список `data_errors`, файл переопределит статус на `ERROR`, а все детали логируются в финальный артефакт отчета.

## Запуск проекта

### Инструкция для macOS / Linux:

1. Активируйте виртуальное окружение:
   source .venv/bin/activate

2. Установите зависимости:
   pip install -r requirements.txt

3. Запустите пайплайн обработки, указав пути к исходным данным и папке для отчетов:
   python main.py --input <путь_к_папке_с_CSV> --output <путь_для_отчетов>

   Пример запуска:
   python main.py --input test_data --output data_reports

### Инструкция для Windows:

1. Активируйте виртуальное окружение:
   .venv\Scripts\activate

2. Установите зависимости:
   pip install -r requirements.txt

3. Запустите пайплайн обработки, указав пути к исходным данным и папке для отчетов:
   python main.py --input <путь_к_папке_с_CSV> --output <путь_для_отчетов>

   Пример запуска:
   python main.py --input test_data --output data_reports

## Запуск тестов
Для запуска всего тестового покрытия, включая тесты бизнес-сценариев и CLI-интерфейса, выполните команду в корне проекта:

* **Для macOS / Linux:**
  python3 -m pytest

* **Для Windows:**
  python -m pytest


## Ограничения и особенности текущей версии

Текущая версия проекта развивает пайплайн в рамках этапа v2 и имеет следующие особенности:

* **Формат данных**: Обрабатываются только файлы с расширением `.csv`.
* **Область валидации строк**: Построчная проверка (`row-level validation`) выполняется **исключительно для полей `date` и `price`**. Для остальных колонок (например, `name`) глубокая валидация значений на данном этапе не реализована.
* **Статичная конфигурация**: Список обязательных колонок жестко зафиксирован в коде конфигурации (`config.py`) и не может быть изменен динамически через аргументы командной строки при запуске.
* **Формат отчетов**: Итоговые отчеты формируются строго в виде плоского текста (`.txt`) и пока не экспортируются в структурированные форматы (JSON, базы данных и т.д.).
* Текущая версия ориентирована на учебный и инженерный сценарий, а не на промышленную высоконагруженную эксплуатацию.
