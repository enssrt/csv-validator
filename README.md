# CSV Validator Pipeline

Утилитарный инструмент на Python для автоматического анализа, валидации и обработки CSV-таблиц с генерацией отчетов.

## Функционал
* Сканирование входной директории и пакетная обработка файлов.
* Валидация структуры CSV (проверка обязательных колонок из конфигурации).
* Логирование результатов и сбор итоговой статистики.
* Автоматическая генерация отчетов по результатам обработки.

## Статусы обработки файлов
* `OK` — Файл успешно прошел валидацию и обработан.
* `ERROR` — Файл содержит ошибки структуры (например, не хватает колонок).
* `NOT_READ` — Не удалось прочитать файл (ошибка доступа или поврежденный файл).
* `REPORT_ERROR` — Файл валиден, но произошла системная ошибка при попытке записать отчет.

## Структура проекта
* `main.py` — Точка входа, запуск пайплайна (`run_pipeline`).
* `source/` — Исходный код логики (обработка папок, валидация, конфигурация).
* `tests/` — Интеграционные и модульные тесты для проверки бизнес-сценариев.
* `requirements.txt` — Список внешних зависимостей проекта.

## Запуск проекта

### Инструкция для macOS / Linux:

1. Активируйте виртуальное окружение:
   source .venv/bin/activate

2. Установите зависимости:
   pip install -r requirements.txt

3. Запустите пайплайн обработки:
   python main.py

### Инструкция для Windows:

1. Активируйте виртуальное окружение:
   .venv\Scripts\activate

2. Установите зависимости:
   pip install -r requirements.txt

3. Запустите пайплайн обработки:
   python main.py

## Запуск тестов
Для запуска тестов и проверки всех бизнес-сценариев обработки (работает на любой операционной системе) выполните команду:
python -m pytest tests/test_main.py

## Ограничения v1

Текущая версия проекта является первой рабочей реализацией пайплайна и имеет несколько ограничений:

* Обрабатываются только файлы формата `.csv`.
* Проверка выполняется только по структуре файла и наличию обязательных колонок.
* Список обязательных колонок задается заранее в конфигурации и не изменяется через интерфейс запуска.
* Проект не выполняет глубокую валидацию значений внутри строк, например корректность дат, чисел или диапазонов.
* Отчеты формируются в текстовом формате и не сохраняются в более сложные структуры, например JSON или базу данных.
* Проект рассчитан на локальный запуск и пока не имеет CLI-интерфейса с аргументами командной строки.
* Текущая версия ориентирована на учебный и инженерный сценарий, а не на промышленную эксплуатацию.
