# README.md

# CSV Validator Pipeline (v2) — [ЗАВЕРШЕНО]

Утилитарный инструмент на Python для автоматического анализа, валидации и обработки CSV-таблиц с генерацией отчетов. Версия v2 полностью завершена, стабилизирована и готова к использованию в качестве зрелого CLI-инструмента.

## Функционал
- Сканирование входной директории и пакетная обработка файлов через интерфейс командной строки.
- Двухуровневая валидация (проверка структуры колонок + построчная проверка значений).
- Логирование результатов и сбор итоговой статистики.
- Автоматическая генерация текстовых отчетов по результатам обработки на диск.

## Статусы обработки файлов
- `OK` — файл успешно прошел валидацию структуры и значений строк.
- `ERROR` — файл содержит ошибки структуры (отсутствие колонок) или значений в строках.
- `NOT_READ` — не удалось прочитать файл (пустой файл или ошибка доступа).
- `REPORT_ERROR` — произошла ошибка при записи файла отчета на диск.

## Структура проекта
- `main.py` — точка входа, парсинг аргументов CLI и запуск конвейера (`run_pipeline`).
- `source/`  
  - `processor.py` — вылизанный, плоский модуль обработки одиночных файлов и папок с поддержкой динамических колонок.
  - `validators.py` — построчная валидация бизнес-параметров (`price`, `date`).
  - `csv_utils.py` / `report_utils.py` — утилиты чтения таблиц и генерации текстовых отчетов в человекочитаемом формате.
  - `config.py` — конфигурационные константы проекта.
- `tests/` — тестовое покрытие, объединяющее 41 тест (unittest и pytest-style сценарии) в один общий поток запуска.

## Архитектура и конфигурация параметров

В текущей версии этапа v2 реализовано гибкое управление обязательными колонками через CLI и внутреннюю конфигурацию.
- **CLI-слой:** `--input` и `--output` являются обязательными аргументами. При их отсутствии программа завершается с кодом `2`.
- **Флаг `--columns`:** позволяет переопределить список обязательных колонок, передав их через запятую при запуске.
- **Fallback:** если `--columns` не передан, используется дефолтный `REQUIRED_COLUMNS` из `source/config.py`.
- **Exit codes:** код `0` — успешное завершение работы пайплайна, код `1` — критическая ошибка инициализации.

## Двухуровневая валидация данных

Конвейер выполняет проверку файлов последовательно. Построчная валидация запускается только после успешной проверки структуры.

### 1. Structural Check
Проверяется наличие всех обязательных колонок, переданных через CLI или взятых из конфигурации по умолчанию. Если структура нарушена, файл получает статус `ERROR`, создается отчет о недостающих колонках, а построчная проверка данных пропускается.

### 2. Row-level Validation
Если структура корректна, запускается построчный анализатор данных по модулю `validators.py`:
- `price` — значение должно быть числовым и строго больше нуля.
- `date` — значение должно соответствовать формату `YYYY-MM-DD` и быть календарно существующей датой.

Если в строках обнаружены ошибки значений, они форматируются в человекочитаемый вид, добавляются в отчет, а файл получает итоговый статус `ERROR`.

## Запуск проекта

### Пример со стандартными колонками
```bash
python3 main.py --input test_data --output data_reports
```
### Пример с кастомными колонками
```bash
python3 main.py --input test_data --output data_reports --columns "id, title"
```
## Запуск тестов
Проект использует полный поток тестирования на базе pytest (всего 41 passed).

# macOS / Linux
```bash
python3 -m pytest
```

# Windows
```bash
python -m pytest
```

# Итоги и ограничения версии v2

- Контур версии v2 полностью закрыт; переход к v3 на текущий момент не начат.
- Обрабатываются исключительно файлы с расширением .csv.

- Итоговые отчеты формируются только в виде плоского текста .txt без выгрузки в сторонние форматы.
